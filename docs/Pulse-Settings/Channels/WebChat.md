# Pulse Web Chat

## Overview

Pulse Web Chat is a website-based chatbot channel that enables organizations to engage visitors, answer questions, capture leads, and provide support directly through their website.

Unlike social channels such as WhatsApp, Instagram, and Messenger, the Web Chat channel includes additional customization capabilities that allow administrators to:

- Configure AI behavior
- Customize the chat widget appearance
- Control visitor engagement
- Design chatbot conversation flows
- Capture lead information
- Automate support and sales journeys

---

## Web Chat Components

The Pulse Web Chat configuration consists of three major sections:

- Chatbot Configuration
- Widget Setup
- Conversation Flow

---

## Chatbot Configuration

The Chatbot Configuration tab controls how the chatbot operates, responds to visitors, creates leads, assigns conversations, and handles support requests.

The Chatbot Configuration tab uses the same channel configuration framework available in other Pulse channels such as:

- WhatsApp
- Instagram
- Messenger

### Shared Configuration Areas

The following settings are configured and managed in the same way as described in the Channels Configuration section:

- Channel Overview
- AI Response Settings
- Lead Creation Settings
- Session Settings
- User Assignment
- Assignment Rules
- Business Hours Behavior
- Compliance & Data Settings

:::note
For detailed information about these settings, refer to the Channels documentation.
:::

---

## Web Chat Specific Notes

Although the configuration structure is the same, the Web Chat channel directly affects website visitor interactions and works together with:

- Widget Setup
- Conversation Flow
- AI Response Engine

:::tip
Changes made to the chatbot channel configuration impact how website conversations are handled, routed, escalated, and converted into leads.
:::
