# Work Schedule Configuration

The **Work Schedule Configuration** section defines the organization's operational calendar.

These settings are used for:

- SLA calculations
- Workflow timing
- Task scheduling
- Business-hour processing

---

![Business Hours](/img/OrganizationSettings/worktime.png)

## Weekly Working Days

Administrators can select the organization's working days.

### Examples

- Monday–Friday
- Sunday–Thursday

:::note
Non-working days are treated as weekends.
:::

### Used In

- Task calculations
- SLA timers
- Workflow scheduling

---

## Business Hours

Administrators can define standard working hours.

### Business Hours Are Used For

- SLA tracking
- Workflow execution windows
- Availability alignment

---

## Holiday Calendar

Administrators can manage organization holidays.

![Holiday Calendar](/img/OrganizationSettings/Holidays.png)

### Available Actions

- Add Holiday
- Edit Holiday
- Delete Holiday
- View Holiday List

### Holiday Impact

Holidays are treated as non-working days for:

- SLAs
- Workflows
- Task scheduling

:::note
Holiday schedules are automatically considered when calculating business hours, SLA deadlines, and workflow execution timing.
:::
